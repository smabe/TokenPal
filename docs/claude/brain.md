# Brain

- `PersonalityEngine`: tiered few-shot examples (anchor lines for recency priming), mood system (6 moods, custom per voice), running gags, guardrails (sensitive apps, late-night tone, cross-franchise filter)
- Three prompt paths: `build_prompt()` (observations), `build_freeform_prompt()` (unprompted thoughts), `build_conversation_prompt()` (user input, single-turn fallback)
- Multi-turn conversation: `ConversationSession` in orchestrator tracks history buffer, `build_conversation_system_message()` + `build_context_injection()` compose the messages array. Config in `[conversation]` section. Observations/freeform suppressed during active session. Session auto-expires after `timeout_s` (default 120s). History capped at `max_turns` pairs (default 10, limited by gemma4's 4-8k context — bump for larger models)
- `ContextWindowBuilder`: per-sense weighted interestingness, acknowledge pattern, composite observations (`_detect_composites()` returns `(line, suppress_senses)` tuples — `snapshot()` two-passes, skipping any sense names a composite supersedes), public API: `active_readings()`, `prev_summary()`, `ttl_for()`. AFK composite ("User is parked on X — no input for N min") fires when sustained-idle + unchanged app + typing-cadence-idle/absent; prepended to the composite list so the 2-line cap can't squeeze it out, and suppresses the raw `idle` line to avoid double-printing
- Topic roulette: `_pick_topic()` in orchestrator, no 3+ consecutive same-topic, focus hints prepended to context. AFK demotion: when `_sustained_idle_active()` AND `activity_level() < 0.15`, any sense whose summary equals its `prev_summary` is multiplied by 0.2 so the sustained-idle reading wins instead of "Ghostty is foreground" twins. Comment-rate gate raises its cooldown ceiling from 90s to 180s while sustained-idle is active so the buddy genuinely pauses, not just retargets
- Change detection: `changed_from` field on `SenseReading`, app_awareness populates "switched from X"
- Pacing: dynamic cooldown (30-90s based on activity), max 8 comments/5min, forced 2-min silence after 3 consecutive, timing jitter. High-signal sense events (git) bypass the gate entirely. `_handle_suppressed_output` funnels every generate-but-can't-emit path through one place that advances `_last_comment_time`, counts the streak, AND calls `_context.acknowledge()` so the gate doesn't loop on the same stale interestingness score forever. Suppressed idle-tool riffs skip that funnel so one bad framing can't starve freeform + drift nudges. Full gate plumbing — emission paths, interestingness scoring, near-dup + prefix-lock guards, forced-silence windows, FilterReason enum, anchor regurgitation — in `docs/commentary-gate.md`
- Freeform thoughts: 15% default, 30% for rich voices (50+ example lines), 45s min gap
- Easter eggs bypass LLM (3:33 AM, Friday 5 PM, Zoom, Calculator)
- Cross-session callbacks: `MemoryStore.get_pattern_callbacks()` detects behavioral patterns (day-of-week skew, first-app-per-session, streaks, startup rituals) from SQLite history. Daily aggregation runs on startup. Callbacks injected into observation prompts as factual one-liners the LLM riffs on. Sensitive apps excluded, results cached per session
- Idle tool rolls: third emission path for quiet stretches. `IdleToolRoller` fires only when the comment gate chose silence, so can't inflate comment rate. NOT gated by `_forced_silence_until` (that silences observation-path dupes, not the dead-air recoverer). Every swallowed riff records `filter_reason` in `idle_tool_fire` telemetry. Full architecture — 20 rules, chain rules, running-bit model, personalization context, warm cache, telemetry — in `docs/idle-tool-rolls.md`. Read before editing `tokenpal/brain/idle_tools.py`, `idle_rules.py`, or the orchestrator wiring.
- Observation enrichment: `tokenpal/brain/observation_enricher.py` splices per-sense one-line descriptions into the snapshot before prompt composition. Handlers today: `_enrich_app_awareness`, `_enrich_process_heat`. Both route through `app_enricher.py` which blocks the first tick for a new app (~3s cap) on a `search()` call, caches in `memory.db` (30d TTL, 24h retry-backoff), gates on `web_fetches` consent, filters sensitive apps. Add new handlers as `_enrich_<sense>` following the same latency budget. Full architecture in `docs/observation-enrichment.md`.
- Utility wedges — companion + executive-function layer from `plans/shipped/buddy-utility-wedges.md`:
  - **Session handoff** (`tokenpal/brain/session_summarizer.py`): every `interval_s` (300s default) the brain compresses the last window of observations into a 2-3 sentence note via the LLM and writes it to the `session_summaries` table. Skip-if-idle guard short-circuits empty windows. On boot, the latest note within `max_lookback_h` (24h default) feeds a new `{previous_session}` slot in observation prompts so the buddy can reference last session's work. Silent writer — never emits bubbles.
  - **Intent drift** (`tokenpal/brain/intent.py`): `/intent <text>` persists a single active-goal row (PK `CHECK(id=1)` + `INSERT OR REPLACE`). When the foreground app matches `distraction_apps` for `drift_min_dwell_s` (5min default), `IntentStore.check_drift()` returns a `DriftSignal`. Drift rides the normal pacing gate and preempts the regular observation path on hitting ticks. Uses `build_drift_nudge_prompt()` template. Intent text passes `contains_sensitive_term` on set (reject, not redact). Auto-expires from drift checks after `max_age_s` (8h default) but stays visible to `/intent status`.
  - **End-of-day summary** (`tokenpal/brain/eod_summary.py`): one-bubble-per-day reflection pulled from `MemoryStore.get_day_digest()`. Auto-fires at startup when yesterday has activity and hasn't been shown (`system/eod_shown` observation row tracks "shown"). `/summary today|yesterday` forces on-demand without marking shown. Silent no-op on empty days. Outside the pacing gate.
  - **Rage / frustration detect** (`tokenpal/brain/rage_detector.py`, opt-in, default off): consumes only `typing_cadence` + `app_awareness` readings. Fires when a rapid/furious burst ends, the user pauses for `rage_post_pause_min_s`-`rage_post_pause_max_s` (60-180s), then switches to a distraction app. NEVER imports `_keyboard_bus` or `pynput` — AST-level test enforces this. Bypasses pacing gate via `changed_from`-style high-signal path. 10min session cooldown. Uses `build_rage_check_prompt()` template, explicitly constrained against scolding or questions.
  - **Proactive git nudge** (`tokenpal/brain/git_nudge.py`): fires when HEAD commit message matches `wip_markers` (wip/tmp/todo/fixup! default), tree is dirty, and `wip_stale_hours` (3h default) have elapsed. Detector hydrates initial state via direct `git` subprocess at startup so a stale WIP that predates the buddy run can nudge without waiting. Git sense now emits `last_commit_ts` + `last_commit_msg` in `data` every poll for the detector to consume. 2h cooldown. `user_present` flag flips True on any reading this tick; false suppresses the nudge.
  - Migration scaffolding: `memory.py` uses `PRAGMA user_version` + a `_MIGRATIONS` list. New tables land via migrations, not the legacy `_SCHEMA` block. Closes parking-lot #31 G2. Two migrations so far: session_summaries (v0→v1), active_intent (v1→v2).
