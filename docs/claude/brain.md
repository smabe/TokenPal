# Brain

- `PersonalityEngine`: tiered few-shot examples (anchor lines for recency priming), mood system (6 moods, custom per voice), running gags, guardrails (sensitive apps, late-night tone, cross-franchise filter)
- Three prompt paths: `build_prompt()` (observations), `build_freeform_prompt()` (unprompted thoughts), `build_conversation_prompt()` (user input, single-turn fallback)
- Multi-turn conversation: `ConversationSession` in orchestrator tracks history buffer, `build_conversation_system_message()` + `build_context_injection()` compose the messages array. Config in `[conversation]` section. Observations/freeform suppressed during active session. Session auto-expires after `timeout_s` (default 120s). History capped at `max_turns` pairs (default 10, limited by gemma4's 4-8k context â€” bump for larger models)
- `ContextWindowBuilder`: per-sense weighted interestingness, acknowledge pattern, composite observations (`_detect_composites()` returns `(line, suppress_senses)` tuples â€” `snapshot()` two-passes, skipping any sense names a composite supersedes), public API: `active_readings()`, `prev_summary()`, `ttl_for()`. AFK composite ("User is parked on X â€” no input for N min") fires when sustained-idle + unchanged app + typing-cadence-idle/absent; prepended to the composite list so the 2-line cap can't squeeze it out, and suppresses the raw `idle` line to avoid double-printing
- Topic roulette: `_pick_topic()` in orchestrator, no 3+ consecutive same-topic, focus hints prepended to context. `_TOPIC_FOCUS_HINTS` covers every ambient sense the picker can choose; without an entry the LLM gets the snapshot but no per-topic steer and tends to fall back to riffing on the foreground app. Add an entry whenever you add a new sense. AFK demotion: when `_sustained_idle_active()` AND `activity_level() < 0.15`, any sense whose summary equals its `prev_summary` is multiplied by 0.2 so the sustained-idle reading wins instead of "Ghostty is foreground" twins. Comment-rate gate raises its cooldown ceiling from 90s to 180s while sustained-idle is active so the buddy genuinely pauses, not just retargets
- Change detection: `changed_from` field on `SenseReading`, app_awareness populates "switched from X"
- Pacing: dynamic cooldown (30-90s based on activity), max 8 comments/5min, forced 2-min silence after 3 consecutive, timing jitter. High-signal sense events (git) bypass the gate entirely. `_handle_suppressed_output` funnels every generate-but-can't-emit path through one place that advances `_last_comment_time`, counts the streak, AND calls `_context.acknowledge()` so the gate doesn't loop on the same stale interestingness score forever. Suppressed idle-tool riffs skip that funnel so one bad framing can't starve freeform + drift nudges. Full gate plumbing â€” emission paths, interestingness scoring, near-dup + prefix-lock guards, forced-silence windows, FilterReason enum, anchor regurgitation â€” in `docs/commentary-gate.md`
- Freeform thoughts: 15% default, 30% for rich voices (50+ example lines), 45s min gap
- Easter eggs bypass LLM (3:33 AM, Friday 5 PM, Zoom, Calculator)
- Cross-session callbacks: `MemoryStore.get_pattern_callbacks()` detects behavioral patterns (day-of-week skew, first-app-per-session, streaks, startup rituals) from SQLite history. Daily aggregation runs on startup. Callbacks injected into observation prompts as factual one-liners the LLM riffs on. Sensitive apps excluded, results cached per session
- Idle tool rolls: third emission path for quiet stretches. `IdleToolRoller` (deterministic, M1+M2) fires only when the comment gate chose silence, so can't inflate comment rate. NOT gated by `_forced_silence_until` (that silences observation-path dupes, not the dead-air recoverer). Every swallowed riff records `filter_reason` in `idle_tool_fire` telemetry. M3 (issue #33) layers `LLMInitiatedRoller` on top, sharing one `FireTracker` with the deterministic path; runs BEFORE it in the idle-eligible block and on hit consumes the tick. Off by default, gated on `TOKENPAL_M3=1` env + `/idle_tools llm_on` config. Full architecture (20 deterministic rules, chain rules, running-bit model, personalization context, warm cache, M3 catalog + cross-path cooldown asymmetry, telemetry) in `docs/idle-tool-rolls.md`. Orchestration of the two rollers lives in `tokenpal/brain/idle_runner.py` (`IdleToolRunner`); Brain calls `await self._idle_runner.maybe_run(snapshot, emitted)` at the post-pass. The runner stays outside the Wedge registry because its gate ("fire when nothing else emitted") doesn't fit BYPASS_CAP / NEEDS_CAP_OPEN / IDLE_FILL. Read before editing `idle_tools.py`, `idle_tools_m3.py`, `idle_rules.py`, `idle_runner.py`, or the orchestrator wiring.
- Observation enrichment: `tokenpal/brain/observation_enricher.py` splices per-sense one-line descriptions into the snapshot before prompt composition. Handlers today: `_enrich_app_awareness`, `_enrich_process_heat`. Both route through `app_enricher.py` which blocks the first tick for a new app (~3s cap) on a `search()` call, caches in `memory.db` (30d TTL, 24h retry-backoff), gates on `web_fetches` consent, filters sensitive apps. Add new handlers as `_enrich_<sense>` following the same latency budget. Full architecture in `docs/observation-enrichment.md`.
- Utility wedges â€” companion + executive-function layer from `plans/shipped/buddy-utility-wedges.md` (subsystem origin) and `plans/shipped/wedge-unification.md` (architectural shape). Today four wedges (rage, git_nudge, drift, freeform) live behind a uniform `Wedge` ABC in `tokenpal/brain/wedge.py` with files in `tokenpal/brain/wedges/`. Each Wedge proposes at most one `EmissionCandidate` per Brain tick; `_select_candidate` ranks proposals by priority and applies a `GatePolicy` (BYPASS_CAP, NEEDS_CAP_OPEN, IDLE_FILL); a single shared `_riff` pipeline runs the LLM, filters, and emits. Vocabulary in `CONTEXT.md`. The `_generate_comment` path stays on Brain (deferred â€” its easter-egg shortcut + tool-calling dispatch + post-emit hooks don't fit the shared pipeline without one-consumer extension points). Detector primitives keep their original homes:
  - **Session handoff** (`tokenpal/brain/session_summarizer.py`): every `interval_s` (300s default) the brain compresses the last window of observations into a 2-3 sentence note via the LLM and writes it to the `session_summaries` table. Skip-if-idle guard short-circuits empty windows. On boot, the latest note within `max_lookback_h` (24h default) feeds a new `{previous_session}` slot in observation prompts so the buddy can reference last session's work. Silent writer â€” never emits bubbles.
  - **Intent drift** (`tokenpal/brain/intent.py` + `tokenpal/brain/wedges/drift.py`): `/intent <text>` persists a single active-goal row (PK `CHECK(id=1)` + `INSERT OR REPLACE`). When the foreground app matches `distraction_apps` for `drift_min_dwell_s` (5min default), `IntentStore.check_drift()` returns a `DriftSignal` which `DriftWedge` (NEEDS_CAP_OPEN, priority 50) wraps as an `EmissionCandidate`. Uses `build_drift_nudge_prompt()`. Cooldown only starts on actual emit (drift's `on_emitted` gates on `success=True`, unlike rage/git_nudge which mark on any post-LLM path). Intent text passes `contains_sensitive_term` on set (reject, not redact). Auto-expires from drift checks after `max_age_s` (8h default) but stays visible to `/intent status`. IntentStore stays on Brain â€” `/intent` slash command + EOD wiring read it independently of drift.
  - **End-of-day summary** (`tokenpal/brain/eod_summary.py`): one-bubble-per-day reflection pulled from `MemoryStore.get_day_digest()`. Auto-fires at startup when yesterday has activity and hasn't been shown (`system/eod_shown` observation row tracks "shown"). `/summary today|yesterday` forces on-demand without marking shown. Silent no-op on empty days. Outside the pacing gate. NOT a Wedge â€” it's a startup one-shot, not a per-tick candidate.
  - **Freeform thoughts** (`tokenpal/brain/wedges/freeform.py`): `FreeformWedge` (IDLE_FILL, priority 5, `acknowledge_emit=False`) wraps the unprompted-thought path. Fires only when the cap is closed and no other wedge proposed. The internal gate (`_should_freeform()`) stays on Brain because it reads broadly across Brain state; the wedge takes it as an injected callable.
  - **Rage / frustration detect** (`tokenpal/brain/rage_detector.py` + `tokenpal/brain/wedges/rage.py`, opt-in, default off): consumes only `typing_cadence` + `app_awareness` readings. Fires when a rapid/furious burst ends, the user pauses for `rage_post_pause_min_s`-`rage_post_pause_max_s` (60-180s), then switches to a distraction app. NEVER imports `_keyboard_bus` or `pynput` â€” AST-level test enforces this. `RageWedge` (BYPASS_CAP, priority 90) wraps the detector and bypasses the comment-rate cap. 10min session cooldown via `on_emitted`. Uses `build_rage_check_prompt()` template, explicitly constrained against scolding or questions.
  - **Proactive git nudge** (`tokenpal/brain/git_nudge.py` + `tokenpal/brain/wedges/git_nudge.py`): fires when HEAD commit message matches `wip_markers` (wip/tmp/todo/fixup! default), tree is dirty, and `wip_stale_hours` (3h default) have elapsed. `GitNudgeWedge` (BYPASS_CAP, priority 85) hydrates initial state via direct `git` subprocess at startup so a stale WIP that predates the buddy run can nudge without waiting. Git sense emits `last_commit_ts` + `last_commit_msg` in `data` every poll for the detector to consume. 2h cooldown. `_user_present` (wedge-internal) flips True on any reading this tick; false suppresses the nudge.
  - Migration scaffolding: `memory.py` uses `PRAGMA user_version` + a `_MIGRATIONS` list. New tables land via migrations, not the legacy `_SCHEMA` block. Closes parking-lot #31 G2. Two migrations so far: session_summaries (v0â†’v1), active_intent (v1â†’v2).
