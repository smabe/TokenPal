# Senses

- `app_awareness` — macOS: Quartz window list (NOT NSWorkspace). Browser titles sanitized (stripped unless matching music player patterns)
- `hardware` — psutil, cross-platform, expressive summaries at high utilization
- `time_awareness` — cross-platform, session duration tracking
- `idle` — pynput, cross-platform, three tiers. Emits on transition (active→idle is silent, idle→active emits a "User returned…" reading) AND a sustained-idle reading every 60s (or on tier-bump) while the user stays AFK. Sustained reading carries `data["event"] == "sustained"` and tier-scaled confidence (0.3/0.5/0.7) so a sharp return still wins topic competition. Returns are debounced: a single phantom mouse/keyboard event opens a `_pending_return` window; the return only commits with ≥2 events or ≥5s of sustained activity. Lone phantoms get marked `_discarded_input_at` and the AFK composite keeps flowing. Touch records the source (`mouse_move`/`mouse_click`/`mouse_scroll`/`keyboard`) for the phantom-discard debug log
- `weather` — Open-Meteo API (free, no key), poll 30min, TTL 1hr, weight 0.0. Opt-in via `/zip` command or first-run wizard. Geocoding + config write in `tokenpal/config/weather.py`
- `music` — macOS: AppleScript for Music.app/Spotify. Checks `System Events` before querying (prevents auto-launch). Track names redacted from logs
- `productivity` — derives from MemoryStore: time-in-app, switches/hour, streaks. MemoryStore injected via `sense_configs`. Filters sensitive app names
- `git` — cross-platform, polls every 15s via async subprocess. Detects new commits, branch switches, dirty state changes. Uses `asyncio.gather` for parallel git calls. High-signal events bypass the commentary gate for immediate reactions. Opt-in via `[senses] git = true`
- `world_awareness` — HN front-page poll (Algolia API, free/keyless). Poll 15min, TTL 1hr. Emits a multi-headline summary: `Top HN: 'A' — 200 pts | 'B' — 50 pts | 'C' — 30 pts` with up to 3 stories, deduped against `_prev_summary`. Each title filtered via `contains_sensitive_term`; sensitives are dropped from the list (the surviving stories still emit). `data["stories"]` carries the full per-story dicts for memory/debug. Opt-in via `[senses] world_awareness = true`. Silent degradation on network failure
- `lobsters` — Lobste.rs hottest-page poll (`/hottest.json`, no key, sets a `User-Agent`). Same shape as `world_awareness`: 15min poll, 1hr TTL, top-3 multi-headline summary `Top Lobsters: 'X' — N pts | …`, sensitive-filter, dedup. Opt-in via `[senses] lobsters = true`
- `github_trending` — GitHub Search API poll (keyless, `?q=created:>{7d_ago}&sort=stars&per_page=3`). 15min poll, 1hr TTL, ~12 reqs/hr per IP — well under the 10/min keyless cap. Summary: `Trending GitHub (last 7d): owner/repo — N★ (lang) — desc | …`. Sensitive-filter runs over `f"{full_name} {description}"`. Opt-in via `[senses] github_trending = true`
- `sun_position` — local compute via `astral` (no network, no key). Reads lat/lon from `config.weather.latitude/longitude` via `load_config()` — same convention `actions/utilities/sunrise_sunset.py` uses, so any user who's run `/zip` gets it for free. 60s poll, emits only on `Phase` transitions (`night`, `dawn`, `golden_morning`, `day`, `golden_evening`, `dusk`). Per-day astral events cached in a single-slot `_cached_date / _cached_events` pair. `_classify` boundaries: golden hour = ±30min around sunrise/sunset; `now == sunrise` enters `golden_morning`, `now == sunset` enters `dusk`. Polar-latitude `ValueError` from `astral.sun()` absorbed (returns None, doesn't crash the brain loop). Tests inject a `now_fn` constructor param to avoid `dt.datetime.now` monkey-patching. Opt-in via `[senses] sun_position = true`
- `battery` — psutil.sensors_battery, transition-only (plugged/unplugged/low/critical/full). Auto-disables on desktops where no battery is detected. Opt-in via `[senses] battery = true`
- `network_state` — online/offline, SSID change, VPN up/down. SSID never emitted raw: sha256[:16] hash used for change detection, friendly labels via `[network_state] ssid_labels`. VPN detection is a best-effort interface-prefix heuristic (`utun`/`tun`/`wg`). Opt-in via `[senses] network_state = true`
- `process_heat` — names the top CPU hog when CPU is pinned >80% for 20s. Aggregates Electron-family renderers under parent name (Slack Helper → Slack). Sensitive-app names replaced with "something's working hard". Emits on trigger + on clear. Opt-in via `[senses] process_heat = true`
- `typing_cadence` — WPM buckets (`idle`/`slow`/`normal`/`rapid`/`furious`) from a rolling 30s keypress window with 2-poll hysteresis to prevent flapping. Emits on bucket transitions, at the 10-minute mark of a sustained `rapid`/`furious` run, and on post-burst silence. Subscribes to `tokenpal/senses/_keyboard_bus.py` — a shared singleton pynput listener that also feeds `idle`. The sense itself never imports pynput and never sees key values; test suite enforces this with a source-grep assertion. Opt-in via `[senses] typing_cadence = true`
- `filesystem_pulse` — watchdog-based activity-burst detection for user-configured root directories. Emits "Activity in <leaf-dir>" when ≥5 file events land in a root within 30s, with a 60s cooldown per root to prevent spam. Default roots when `[filesystem_pulse] roots` is empty: `~/Downloads`, `~/Desktop`, `~/Documents` (platform-aware via `Path.home()`). Excludes high-noise dirs (`node_modules`, `.venv`, `.git`, `__pycache__`, `build`, `dist`, `target`, `.next`, `.tox`, `.pytest_cache`, `.mypy_cache`, `.ruff_cache`, `Pods`, `DerivedData`). Privacy: summaries include only leaf directory names — never full paths, never filenames. Opt-in via `[senses] filesystem_pulse = true`

## Shared utilities

- **`tokenpal/util/http_json.py`** — keyless HTTP-JSON helper used by every network sense (HN, Lobsters, GitHub trending) and the `web_search` backends. Returns parsed JSON or `None`; never raises. Sets a `TokenPal/1.0` UA. New polling senses MUST route through this rather than hand-rolling `urllib.request`.
- **`tokenpal/util/text_guards.truncate_ellipsis(text, max_chars)`** — single-source truncate-with-`…` for headline/description display. Used by lobsters, world_awareness, github_trending. Reuse rather than re-rolling `text[:n] + "…"`.

## Adding a new ambient news sense

The lobsters / world_awareness / github_trending shape has stabilized as a pattern:

1. `tokenpal/senses/<name>/_client.py` — `fetch_top_X(limit: int) -> list[X]`. Use `http_json` from `tokenpal/util/http_json.py`. Return parsed dataclass instances. Walrus + `if (x := _parse(item))` in the comprehension.
2. `tokenpal/senses/<name>/sense.py` — `poll()` does: `fetch → list-comp filter via contains_sensitive_content_term → if empty return None → format each via truncate_ellipsis → " | ".join → dedup against _prev_summary → emit`. Mirror an existing sense.
3. `tokenpal/config/schema.py` — add field to `SensesConfig` (default `False`).
4. `config.default.toml` — add line under `[senses]` with one-line opt-in comment.
5. `tokenpal/brain/orchestrator.py` `_TOPIC_FOCUS_HINTS` — add hint so the topic picker knows to riff on it. Phrase as "Pick one of the X shown and react to it" for multi-item summaries.
6. `tests/test_senses/test_<name>.py` — mirror a sibling test file. `enabled_config` fixture comes from `tests/conftest.py` automatically; don't redefine.

A base class for these has been considered three times and rejected each time: per-sense divergence (formatter, sensitive-text source, data-dict shape) is larger than the shared body, and threading `_prev_summary` state through a free function is uglier than the current duplication. Don't extract until that math changes.
