# UI

- **Default overlay: Qt** (`tokenpal/ui/qt/overlay.py`, PySide6). Frameless always-on-top dangleable buddy + tray/menu-bar icon + hideable chat window. Textual (`tokenpal/ui/textual_overlay.py`) is the terminal fallback — auto-selected when PySide6 is missing, `TOKENPAL_HEADLESS=1`, or no `DISPLAY` / `WAYLAND_DISPLAY` on Linux. `resolve_overlay` in `tokenpal/ui/registry.py` logs the specific fallback reason at INFO; bad names still raise. Console + tkinter overlays remain as additional fallbacks
- **Adapter seam**: every overlay implements `tokenpal/ui/base.py:AbstractOverlay`. Abstract lifecycle methods (setup/run_loop/teardown/schedule_callback/show_buddy/show_speech/hide_speech); no-op defaults for the capability surface (`set_mood`, `load_voice_frames`, `clear_voice_frames`, `set_voice_name`, `set_environment_provider`, `set_chat_persist_callback`, `toggle_chat_log`, 5 modal openers). `tests/test_ui_adapter_contract.py` asserts every registered overlay satisfies the full surface, so `app.py` never `hasattr(overlay, …)`-probes
- **Qt architecture** (read `docs/qt-frontend.md` before editing anything under `tokenpal/ui/qt/`): `qt/overlay.py` owns a `_UIBridge(QObject)` with a single `dispatch = Signal(object)` wired via `QueuedConnection`; every brain-thread adapter call wraps work in a 0-arg lambda and marshals onto the Qt main thread. Pre-setup calls buffer in `_pending_post` and drain on mount. Widgets: `BuddyWindow` (rigid-pendulum drag physics pivoted at cursor grab point, `qt/physics.py:PendulumSimulator`; edge-dock within 20 px of screen edge), `SpeechBubble` (30 ms typing animation, wrap cache), `ChatDock` (frameless translucent input + status strip; glass-pill `QLineEdit` is the only focusable child so the strip never steals focus; `/` routes to command callback, else freeform) + `ChatHistoryWindow` (frameless translucent scrollable log with top drag handle + bottom Hide button + bottom dock slot; starts hidden, tray / `toggle_chat_log` / F2 toggles it; shared drop-shadow + glass helpers in `qt/_text_fx.py`; per-glyph shadow applied to the `QTextBrowser`'s viewport, not the frame — `QAbstractScrollArea` paints into the viewport). Dock mount state machine: `_update_dock_placement` / `_apply_dock_mode(mode)` with `mode ∈ {floating, embedded, hidden}` driven by `_buddy_user_visible` + `_history_user_visible` (explicit user-intent, separate from Qt `isVisible()` which lies on macOS). Buddy shown → dock floats under him; buddy hidden + history shown → dock embeds in history's bottom slot, left edge flush with log; both hidden → dock hidden. Neither toggle touches the other's state. `BuddyTrayIcon` click only pops the menu (no click-to-toggle — macOS menu-bar footgun), modals in `qt/modals.py` (ConfirmDialog / SelectionDialog), `qt/options_dialog.py` (full /options parity), `qt/cloud_dialog.py` (Anthropic/Tavily/Brave + refine cap, deep-mode guardrail), and `qt/voice_dialog.py` (switch/off/train/regenerate/ascii/cloud_classifier). `_OneShotCallback` mixin guarantees modal `on_result` fires exactly once across every accept path — never listen to `QDialog.finished` alongside `accepted`/`rejected`
- **Qt threading rule**: never call widget methods from the brain thread. Always `_post(lambda: ...)` or `schedule_callback(fn, delay_ms)`. The bridge + queued connection marshal automatically; `schedule_callback(fn, delay>0)` composes a `QTimer.singleShot` on the UI thread
- **Qt font config** (Qt-only): two `FontConfig` blocks in `[ui.chat_font]` / `[ui.bubble_font]` (family / size_pt / bold / italic / underline). Chat window gets live `Cmd/Ctrl +/-/0` shortcuts via `QKeySequence.StandardKey.ZoomIn/ZoomOut` (auto-maps per platform), clamped `[8, 48]`, reset = 13pt baseline. Dialog + shortcut paths both flow through `QtOverlay.set_chat_font` / `set_bubble_font` (live apply) + `set_font(section, cfg)` writer (persist). `SpeechBubble.apply_font_config` re-runs layout and invalidates the wrap cache. Loader registration in `_NESTED_FIELDS` is load-bearing — without it the TOML section stays a dict and breaks attribute access. Full wiring in `docs/qt-frontend.md` § Font sizing and style
- **Qt platform polish** (`tokenpal/ui/qt/platform.py`): macOS `NSApplication.setActivationPolicy_(NSApplicationActivationPolicyAccessory)` hides the Dock — **must** run AFTER QApplication construction; ordering warning in `qt/overlay.py:setup()` and `qt/platform.py`. Linux Wayland logs a one-line "always-on-top is compositor-dependent" at INFO. Hi-DPI rounding policy set to `PassThrough` in `qt/__init__.py:ensure_qapplication` before QApplication construction
- **Pendulum drag physics** (`qt/physics.py:PendulumSimulator`): rigid pendulum pivoted at the cursor's grab point on the art. Grab the head, dangle normally; grab a foot, dangle upside-down; grab anywhere in between and swing around that point. Semi-implicit Euler at 60 Hz. ODE sums five cursor-driven forcing terms (all fade cleanly to the background once body hits rest): (1) gravity `−g·sinθ/L`, (2) velocity-based wind-drag `drag·v_tangential/(L·m)` for steady tilt during a linear drag, (3) acceleration-based yank `yank·a_tangential/L` for whip impulses, (4) angular damping (scales down at high ω so a twirl can sustain), (5) circular-motion coupling `coupling·ω_cursor` — detects the cursor's angular rate around its running-mean position via swept-area cross-product over a 0.4 s window, EMA-smoothed so hand-jitter doesn't sag the drive. Wind-drag + yank + gravity fade out during orbit (`spin_lockout_rate=4`) so a big circle twirl doesn't get parasitic sine-wave oscillation. On release, pivot re-anchors to head preserving body's world position (head moves to the body's current head position, not the cursor) — no visual pop, gravity then rotates him upright. `DangleSimulator` (the old 2D spring) is still in the file but unused; can be removed in a follow-up. Auto-sleep same contract as before; `BuddyWindow` stops/restarts its 60 Hz timer so a resting buddy burns zero CPU. Debug HUD + log available via `TOKENPAL_PHYSICS_DEBUG=1`
- **Installers**: `--headless` (`install-macos.sh`, `install-linux.sh`, `setup_tokenpal.py`) / `-Headless` (`install-windows.ps1`) drops the `desktop` extra. Default install includes PySide6
- Textual still works and is the fallback path. Section below documents Textual specifics:
- Layout: horizontal split — buddy panel (left) with header/speech/buddy/input/status, scrollable chat log (right). Buddy/input/status are wrapped in `#buddy-footer` (height: auto) so the bubble can never push them off-screen; speech + spacer share `#speech-region` (1fr) which absorbs leftover space
- Chat log shows all buddy comments (observations + conversation) and user messages, with timestamps and voice/user labels separated by `──────` dividers. Uses `Static(markup=True)` with Rich markup; all text escaped via `_esc_markup`, lines accumulated in `_chat_log_lines` (capped at 500). URLs from `/ask` and the `research` tool's citations render as clickable links via `[@click=app.open_chat_link("idx")]` Textual actions (not OSC 8 -- works in any terminal). Auto-hides when terminal width drops below `buddy.max_frame_width() + _BUDDY_PANEL_PADDING + _CHAT_LOG_MIN_SPACE` (F2 always overrides). `#buddy-panel.min_width` is set dynamically to `max_frame_width() + 4` so frames never wrap, and `#buddy { text-wrap: nowrap; overflow-x: hidden }` crops rather than wraps if the panel is forced narrower
- Chat log persistence: `_append_log` write-throughs to the `chat_log` SQLite table. `MemoryStore` owns the cap (`set_chat_log_max_persisted`) with an in-memory `_chat_log_count` primed at setup so the hot path is INSERT + maybe DELETE (no per-insert COUNT). Trim fires above `max_persisted * 1.5` and rewrites surviving `@click` indices + prunes `_link_urls`. Startup hydrates up to `[chat_log] hydrate_on_start` entries via `overlay.load_chat_history` (stashes then drains in `on_mount`). `_format_chat_ts` shared live/hydrated: today→`%I:%M %p`, older→`%b %d %I:%M %p`. Opt out via `[chat_log] persist = false` (cap=0 at startup)
- Modal stacking guard: `TokenPalApp._modal_already_active()` walks `screen_stack` for any `ModalScreen` and every `on_open_*_modal` handler returns early when one is already live. Launcher buttons (OptionsModal → Cloud/Senses/Tools) still work because `dismiss()` pops the current modal before the callback reposts
- Status bar: `weather | voice·mood | server | model | app | music | spoke Xs ago` — the first four fields are the contract the Qt dock renders; Textual shows all seven. Mood is color-coded in Textual
- Keyboard shortcuts (Textual only; Qt uses tray menu + chat window input): F1=/help, F2=toggle chat log, F3=/options, F4=/voice, F7=toggle buddy (hides buddy panel so chat log goes full-width — used with Shift+drag on Windows Terminal to cleanly select+copy chat text without buddy ASCII art interleaving), Ctrl+L=/clear, Ctrl+C=quit
- Speech bubbles: typing animation via `set_interval(0.03)`. SpeechBubbleWidget is a `VerticalScroll` wrapping an inner Static so long bubbles get a scrollbar. Three render tiers picked by `_choose_bubble_variant` and re-picked on resize: bordered (requires `region_w >= _MIN_BORDERED_REGION_WIDTH = 36` and fits in region_h, max_width clamped accounting for `_SPEECH_SCROLL_PADDING = 4`), borderless (plain wrapped text, no border/tail, full region width), or hide. Suppressed bubbles park in `_pending_bubble` and promote via `show_immediate` (no replay-from-zero typing) when the terminal grows. Incoming bubbles arriving mid-animation queue (cap `_MAX_BUBBLE_QUEUE = 3`) so observations don't cut off conversation replies. Chat log always logs every bubble on arrival, so suppression never silently drops content. Voice ASCII art markup is healed at load time in `ascii_renderer._fix_markup` — Rich-only color names (silver/gray/etc) get remapped to hex so Textual's stricter `Style.from_rich_style` doesn't crash with `MissingStyle`
- Thread safety: brain→UI via typed `Message` subclasses + `post_message()`, never `call_from_thread`
- Voice-specific ASCII art: LLM-generated Rich-markup frames (idle, idle_alt, talking) with 4s blink animation
- Buddy environment layer (`tokenpal/ui/buddy_environment.py` + `ascii_props.py`): Textual-only. Sibling `ParticleOverlay` renders weather props + Newtonian particles (dust/rain/snow/lightning/steam), cap `PARTICLE_LIMIT=50`. Brain pushes an `EnvironmentSnapshot` at 1 Hz via `set_environment_provider`; overlay's 10 Hz tick reads the buffer, never touches brain. Sensitive-app suppression freezes motion; sustained-idle slows it. Hot weather (≥85°F, unit-aware) emits sweat-bead steam. Extend: new `Kind` + spawn helper in `buddy_environment.py`, sprite in `ascii_props.py` (hex colors only), test in `tests/test_buddy_environment.py`.
- Qt weather overlay (`tokenpal/ui/qt/weather.py`): parallel Qt implementation. `SkyWindow` (top-right, frameless + translucent + `WA_TransparentForMouseEvents`) paints sun/moon/overcast double-stack/drifting cloud/lightning alpha strobe/shooting stars. `BuddyRainOverlay` tracks `BuddyWindow.buddy_occlusion_rect_world()` and paints drops/splashes/snow-dust that overlap the buddy; contact uses the tight art-frame AABB via `BuddyWindow.world_to_art` + `art_bounds`. Single `WeatherSim` owns all particles (30 Hz tick on the sky window) so drops don't tear at the sky↔overlay seam. `sensitive_suppressed` CLEARS particles here (intentional divergence from Textual's freeze — a frozen drop is more attention-drawing than none). Lightning is a bounded sky-alpha strobe (0.45 max alpha, ≥8 s min gap, ~160 ms total), NOT a port of Textual's bolt. Shooting stars replace Textual's full starfield on clear nights (Poisson 30–120 s, trajectory passes through the moon). Drops land visually on the buddy but do NOT feed back into `PendulumSimulator`.
