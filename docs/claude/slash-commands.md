# Slash Commands

- `/help`, `/clear`, `/mood`, `/status`, `/chatlog`
- `/math <expr>` ŌĆö evaluate an arithmetic expression (+, -, *, /, //, %, **). Expression length capped, exponent capped. Bypasses the LLM entirely
- `/senses [list|enable <name>|disable <name>]` ŌĆö inspect + toggle sense flags in config.toml. Writes via `tokenpal/config/senses_writer.py`. Senses are resolved once at startup, so the command always reminds the user to restart
- `/wifi label <friendly name>` ŌĆö read the current SSID, hash it (sha256[:16]), and upsert `[network_state] ssid_labels` in config.toml. Restart required to apply. Raw SSID never persisted
- `/watch [list|add <path>|remove <path>]` ŌĆö manage `filesystem_pulse` root directories. `list` shows effective roots (configured or defaults). `add`/`remove` upsert absolute paths in `[filesystem_pulse] roots`. Restart required to apply
- `/options` ŌĆö umbrella settings modal (also on F3). Sections: **Chat history** (digit-sanitized `max_persisted` input + "Clear history now" button), **Chat font** + **Speech bubble font** (Qt only ŌĆö `QFontComboBox` family picker over OS-installed fonts, `QSpinBox` size [8, 48], bold/italic/underline checkboxes, live preview label; `_FontGroupWidgets.baseline` snapshots state post-construction so save only fires on real changes), **Server** (one row per known server as `label ŌĆö url ŌĆö model`, active row bold/disabled, plus a CustomŌĆ” URL input ŌĆö clicking a row re-dispatches through `_cmd_server(f"switch <url>")` so persistence + model-restoration stay in one place), **Models (on this server)** (one row per model advertised by `llm.available_models`, populated by the `/v1/models` probe cached in `HttpBackend._try_connect`; clicking re-dispatches through `_cmd_model(name)`). Plus launcher buttons that dismiss-then-open the existing Cloud / Senses / Tools / Voice modals (`_open_cloud_modal` / `_open_senses_modal` / `_open_tools_modal` / `_open_voice_modal` in `app.py`). Modal is in `tokenpal/ui/options_modal.py`; result dataclass carries `navigate_to`, `switch_server_to`, `switch_model_to`, `set_chat_font`, `set_bubble_font`. Qt-only keyboard shortcuts on the chat window: `Cmd/Ctrl +/-/0` bump/reset chat font size live (persisted via `set_chat_font_persist_callback`). `/clear` (Ctrl+L) also wipes persisted rows ŌĆö the modal's "Clear history now" is the discoverable version
- `/idle_tools [list|on|off|enable <rule>|disable <rule>|roll <rule>]` ŌĆö inspect + toggle the idle-tool roller rules. See `docs/idle-tool-rolls.md`. Restart required for toggle changes.
- `/gh [log|prs|issues]` ŌĆö GitHub integration. Runs git/gh CLI in a daemon thread, logs raw output to chat log, then feeds it to the brain so the buddy comments in character
- `/model [name|list|pull|browse]` ŌĆö model management
- `/voice [train|switch|list|off|info|finetune|finetune-setup|regenerate|ascii|import]` ŌĆö voice management. Bare `/voice` opens the voice dialog on Qt (`qt/voice_dialog.py`) and Textual (`VoiceModal`); console/tkinter fall back to the usage string. Modal + slash share one dispatch path via per-subcommand helpers (`_voice_list`/`_voice_info`/`_voice_off`/`_voice_switch` + `_start_voice_*` thread launchers) ŌĆö when adding a new subcommand, wire BOTH the slash branch in `_handle_voice_command` AND a modal button so the two never drift. `regenerate` refreshes all LLM-backed assets (~60s, confirm-gated); `ascii` refreshes only the three ASCII-art frames (idle/idle_alt/talking) so you can iterate on art without re-baking persona/moods/etc.
- `/server [status|switch]` ŌĆö server connection
- `/zip <zipcode>` ŌĆö set weather location (geocodes via Open-Meteo, writes to config.toml)
- `/ask <question>` -- web search via DuckDuckGo Instant Answer + Wikipedia REST fallback (free, keyless). Gated by the `web_fetches` consent category (`~/.tokenpal/.consent.json`); run /consent to grant. Results filtered through `contains_sensitive_term`, wrapped in `<search_result>` delimiters, fed to brain via `submit_user_input()` -- conversation-session follow-up auto-opens. Source URL displayed as clickable link in chat log. `/ask` is human-typed only ŌĆö the `search_web` LLM tool was removed because the model reached for it on deep questions where it should have used `research`; only `research` is LLM-callable now. Search backend abstraction in `tokenpal/senses/web_search/client.py` (BackendName Literal, `_clear_conversation()` zeros history refs on session timeout)
- `/research <question>` ŌĆö plan ŌåÆ search ŌåÆ fetch ŌåÆ synth ŌåÆ validate pipeline, JSON-structured synth, substring-grounded claim validation. Multi-backend search: DDG (free default), Tavily, Brave, HN, StackExchange. Planner picks backend per query; `_resolve_backend` falls back safely on hallucinated/unconfigured names. Thin-pool top-up: any backend miss under `_THIN_POOL_THRESHOLD=3` triggers a DDG refetch, deduped by canonical URL (tracking params stripped). Per-run telemetry dual-sinks to session log + chat transcript. Tool-invoked research writes to `research_cache` via `MemoryStore.research_cache_key` so `/refine` finds runs regardless of entry point. Full architecture in `docs/research-architecture.md` ŌĆö read before editing `tokenpal/brain/research.py`, `actions/research/fetch_url.py`, or the conversation system prompt.
- `/refine <follow-up>` ŌĆö re-analyze the most recent `/research` run's cached sources against a new question. Cloud-only (`/cloud anthropic enable` required). Refuses after `/cloud deep|search` runs because those server-side modes don't cache excerpts ŌĆö refuse message names both modes so users know which toggles to flip off. Source cache lookup via `MemoryStore.get_latest_research` with max-age gate. Wiring in `tokenpal/brain/orchestrator.py` + `tokenpal/app.py`.
- `/followup <question>` ŌĆö cheap follow-up on the most recent cloud `/research` via Anthropic prompt caching. Replays the saved message history with a `cache_control: ephemeral` breakpoint on the prior assistant turn so the cached prefix bills at 10% of fresh tokens (~$0.02-0.05 per follow-up vs ~$0.20 fresh). Works for all three cloud modes (synth/search/deep) because it uses whatever message history the initial run produced. One `FollowupSession` lives on `Brain._active_followup_session`; overwritten on next `/research`, 15min TTL, 5 follow-ups max per session (`ResearchConfig.followup_*` config knobs). The conversation LLM also calls `research_followup` automatically via tool use ŌĆö personality rule 9 gates the escalation so it only fires when the prior `<answer>` genuinely doesn't cover the follow-up. `/followup` slash is the explicit override. Anthropic's cache has a **hard 4-breakpoint-per-request limit**: `_apply_cache_breakpoint` in `tokenpal/llm/cloud_backend.py` strips any accumulated markers before adding one ŌĆö essential past followup #4 because otherwise stale `cache_control` from prior rounds accumulates in `session.messages` and 400s. Min cacheable prefix: 2048 tokens (Sonnet 4.6), 4096 (Haiku 4.5 / Opus 4.7) ŌĆö below that, Anthropic silently skips caching and `cache_creation_input_tokens=0`. Full architecture + live-verify script at `scripts/verify_followup_live.py` in `docs/research-architecture.md` ┬¦Follow-ups on cloud research.
- `/intent <text>` ŌĆö set an ambient goal; buddy nudges on drift into configured distraction apps. `/intent status` shows current intent + age; `/intent clear` removes it. See `plans/shipped/buddy-utility-wedges.md` Phase 2 for drift rules.
- `/summary [today|yesterday]` ŌĆö on-demand end-of-day reflection bubble. Yesterday also fires automatically once on first startup of a new local day.
- `/intent` and `/summary` also respond to plain-English phrasings ("remind me to X", "what am I working on", "give me a summary", "what did I do yesterday"). Regex matcher in `tokenpal/nl_commands.py` hooks into `_on_user_input`; on hit it re-dispatches through the slash pipeline so behavior is identical to typing the slash.
- `/cloud` ŌĆö two-level dispatcher: `/cloud <backend> <action>` where backend Ōłł {`anthropic`, `tavily`, `brave`}. Legacy flat form (`/cloud enable/disable/forget/model/plan/deep/search`) continues to dispatch to `anthropic` with a deprecation log line. All keys stored at `~/.tokenpal/.secrets.json` (0o600), never in `config.toml`; fingerprint-only in status output.
  - **Anthropic** (`/cloud anthropic [status|enable <key>|disable|forget|model <id>|plan on|off|search on|off|deep on|off]`) ŌĆö opt-in Anthropic-backed synth for `/research` only. Fallback to local synth on any `CloudBackendError`. Three cloud-web modes (deep wins over search when both are on): **`/cloud search on`** (Sonnet+ only) attaches `web_search_20260209` only, Sonnet drives the search loop but never fetches full pages (~$0.15-0.25/run). **`/cloud deep on`** (Sonnet+ only) adds `web_fetch_20260209`, tool-loop re-bills full page content each step ($1-3/run on review-heavy queries; cost warning on activation). Deep/search sources have no cached excerpts, so `/refine` refuses after a deep/search run. See `plans/shipped/claude-api-research-synth.md` and the "Deep mode" subsection of `docs/research-architecture.md`.
  - **Tavily** (`/cloud tavily [enable <key>|disable|forget|status]`) ŌĆö LLM-optimized search with preloaded article content. When enabled, `/research` uses Tavily as the default search backend; results skip the local fetch+extract stage. Tavily's free tier is ~1,000 credits/month; `/research` uses 2 credits per query on `advanced` depth (configurable in `[cloud_search]`). Thin-pool top-up from DDG when Tavily returns <3 sources.
  - **Brave** (`/cloud brave [enable <key>|forget|status]`) ŌĆö free-tier web search (2k queries/month). No runtime flag ŌĆö presence of a key = active. Routed to when the planner picks `backend: "brave"`. Key also readable from `TOKENPAL_BRAVE_KEY` env var.
