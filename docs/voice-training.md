# Voice Training

## Commands
- `/voice train <wiki> "<character>"` — scrape wiki, generate structured persona card, greetings, offline quips, custom mood names, structure hints via 5 parallel Ollama calls, then run ASCII art generation serially afterward (the art prompt consumes the fresh persona).
- `/voice regenerate [name|--all]` — re-run *every* LLM-backed generator for an existing profile: persona, anchor_lines, banned_names, greetings, offline_quips, moods, structure_hints, and ASCII art. Preserves `lines`, finetune metadata, `source`, and `created`. ~60s per voice on Qwen3-14B. Omit the name to regenerate the active voice.
- `/voice ascii [name|--all]` — re-run *only* the ASCII art generator. Use when the text data is good but the three frames (idle, idle_alt, talking) came out off-model. Much faster than a full `regenerate` because it skips the five parallel generators. Hot-swaps the refreshed frames into the running `PersonalityEngine` if the active voice was the target.

## When to use `ascii` vs `regenerate`
- Wrong proportions, misaligned mouth frames, broken Rich-markup color, mirrored face: `ascii`
- Voice feels off-character, moods wrong, catchphrases stale: `regenerate`
- Model drifted so hard the underlying lines feel wrong: `/voice train` again

`ascii` is safe to re-run repeatedly. Each call rerolls at two temperatures (0.8 → 0.6) and saves whatever the second attempt produces even if it didn't pass `_frames_look_usable` — so you'll always have *something* to look at, and you can keep hitting `/voice ascii` until a roll lands.

## Persona Format
Structured VOICE/CATCHPHRASES/NEVER/WORLDVIEW card using "You [verb] [pattern]" instructions, not personality adjectives. Generated by `_generate_persona()` in `train_voice.py` with 25 sample lines, max_tokens=500.

Example:
```
VOICE: You speak in short, excited bursts. You use math words as exclamations.
CATCHPHRASES: "Mathematical!", "Algebraic!", "What the cabbage?!"
NEVER: Modern internet slang ("dumpster fire", "lowkey"). Cynicism or sarcasm.
WORLDVIEW: Everything is an adventure. Helping people is what heroes do.
```

## Anchor Lines
Top 150 most distinctive dialogue lines scored at training time via `_score_line()` heuristic (length sweet spot, catchphrase presence, personality markers). Used for tiered few-shot sampling at runtime: 60% from anchor pool, 40% from general pool, 2 anchors placed last for recency effect.

## Cross-Franchise Guardrails
- `banned_names`: characters from OTHER franchises, auto-derived from source URL via `_derive_banned_names()`
- Response filter in `PersonalityEngine` suppresses any output mentioning a banned name (e.g. Bender mentioning Finn)
- Franchise mapping in `_FRANCHISE_CHARACTERS` dict covers Adventure Time, Futurama, Regular Show

## ASCII Art Frames
Each voice ships three Rich-markup frames stored on the profile as `ascii_idle`, `ascii_idle_alt`, and `ascii_talking`. The Textual overlay cycles idle/idle_alt on a 4-second blink timer and swaps to talking while a speech bubble is animating.

Generation pipeline (`_generate_ascii_art` in `train_voice.py`):
1. One Ollama call seeded with the first 300 chars of the persona plus a detailed requirements block (10 lines tall, 20-24 cols wide, hex-only Rich colors, half-block glyphs, signature features). `max_tokens=1200`.
2. Two attempts at temperatures `(0.8, 0.6)`. If the first at 0.8 yields a usable frame set, return it immediately; otherwise retry once at 0.6. If both fail `_frames_look_usable`, return the second attempt anyway as a best-effort fallback — lets the user see what the model produced and decide whether to `/voice ascii` again.
3. `_frames_look_usable` is deliberately loose — it only requires each of the three frames to have ≥4 non-empty lines. Stricter checks (markup roundtrip, character count) were tried and produced too many false rejections on otherwise-fine art.
4. `_parse_ascii_frames` splits on `IDLE:`, `IDLE_ALT:`, and `TALKING:` headers (case-insensitive, trailing colon optional), strips ```-fences, and pads frames shorter than 10 lines with blanks. If `idle_alt` came back empty or identical to idle, copies idle and applies a minor eye-swap tweak so the blink still animates.
5. At overlay load time, `ascii_renderer._fix_markup` heals Rich-only color names (`silver`, `gray`, etc.) into hex codes because Textual's stricter `Style.from_rich_style` crashes with `MissingStyle`. The generator prompt already forbids named colors, but older v1 profiles sometimes slipped them through.

`save_profile` writes the full JSON back to `<voices_dir>/<slug>.json` via a single `write_text` — not atomic. A crash mid-write can truncate the profile, so don't run `/voice ascii --all` over SSH with a flaky connection. Single-voice runs finish fast enough that this is rarely a concern in practice.

## Voice Priming
The voice reminder at the generation point uses actual catchphrases instead of meta-instructions:
- Old: `(Remember: you are Finn. Stay in character. No emojis.)`
- New: `(Finn's style: "Mathematical!", "Algebraic!", "Shmowzow!")`

This exploits the recency effect — the model's next-token prediction is primed by character-specific tokens.

## Shared Utilities
`voice_profile.py` exports: `franchise_from_source()`, `parse_catchphrases()`, `FANDOM_NAMES` — used by both training pipeline and runtime.

## Profile Storage
- Profiles saved to `~/.tokenpal/voices/<slug>.json`, auto-activated in config.toml
- Voice persona replaces default TokenPal identity (not appended)
- Custom moods: pipe-delimited prompt, `_parse_custom_moods()` regex parser
- Version field: v1 = old one-sentence persona, v2 = structured card

## Files
- `tokenpal/tools/train_voice.py` — training pipeline, persona generation, anchor scoring
- `tokenpal/tools/voice_profile.py` — data model, shared utilities
- `tokenpal/brain/personality.py` — runtime prompt building, response filtering
